### Key Technical Insight
**Lifecycle vocabulary mismatch costs a deploy.** I implemented "stay on Today until delivered" literally (HANDLED = `{delivered, wrapped, given}`), but the user's "delivered" meant *physically delivered to the recipient* — what we call `given`. The lifecycle has two distinct delivery events: package-arrives-at-home (`delivered`) and gift-handed-over (`given`). When the spec uses domain language that overlaps a code enum, double-check which event is actually terminal before assuming.

Two other lessons:
1. **Codex hostile review caught real P0s the build was hiding** — cross-source data corruption (tracking review endpoint accepting Amazon run IDs) and Amazon Logistics mis-registration via USPS-fallback slug. Neither would have triggered tests; both would have appeared as "subtle production weirdness". Worth the 5 minutes.
2. **SQLite `PRAGMA foreign_keys = OFF` must run outside a transaction.** Inside one it's a no-op. The migrate.ts wrapper now disables FK before `db.transaction()` opens BEGIN, then re-enables and `foreign_key_check`s afterward. This is what made the 014 recreate-table migration safe.
