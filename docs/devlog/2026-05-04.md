### Key Technical Insight
Shippo gates **live keys** behind a manual approval flow ("Request live key" button on Developer Keys page). Test keys are self-service but only return mock data for synthetic tracking numbers (`SHIPPO_DELIVERED`, `SHIPPO_TRANSIT`, `SHIPPO_PRE_TRANSIT`, `SHIPPO_RETURNED`, `SHIPPO_FAILURE`). Real USPS/UPS/FedEx numbers will 404 against a test key.

The provider-agnostic naming refactor (column `tracking_provider_id`, function `registerWithProvider`, env var `SHIPPO_API_KEY`) means future swaps (e.g., back to AfterShip if their pricing changes, or to EasyPost) are localized to `src/lib/server/tracking.ts` plus an env var change.
