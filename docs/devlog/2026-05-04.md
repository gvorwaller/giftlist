### Files Modified

- `.env.example` — documented URL-token model and live-key approval flow

### Key Technical Insight
Shippo's three webhook security models per [docs](https://docs.goshippo.com/docs/tracking/webhooksecurity):
1. **IP allowlist** — restrict to Shippo's IP ranges (operationally fragile)
2. **URL token** — append `?token=...` to webhook URL (self-service)
3. **HMAC-SHA256** — `t=...,v1=...` signature in `shippo-auth-signature` header — but only via account-manager request with up to 10 business day SLA

The original implementation (commit `fddc4e2`) assumed HMAC was self-service. Token model is functionally equivalent at single-tenant scale: keep the URL secret and the token IS a bearer credential. Rotate by regenerating + editing the Shippo dashboard URL + restarting pm2.
