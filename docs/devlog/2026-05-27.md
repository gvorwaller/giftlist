# 2026-05-27 Devlog

## 20:12 EST - Bug triage: restore 500, missing gift, undo-return, skip confirm

### Summary
Four UX bugs fixed in one session — a 500 on gift restore, a disappeared self-package, a missing undo-return capability, and an accidental-skip risk on the Today page.

### What Was Done

**Bug 1: 500 on Restore (unique constraint)**
- Restoring an archived gift with `(order_pk, line_item_index)` matching an active gift hit the `gifts_order_lineitem_active` partial unique index, surfacing a raw 500
- Root cause: duplicate gifts from pre-matcher-fix Amazon imports shared the same order slot
- Fix: `archiveGift()` now clears `line_item_index` on restore — it's internal import bookkeeping with no user-facing meaning, so the constraint is silently resolved

**Bug 2: Occasion year missing space ("Graduation2026")**
- Svelte 5 strips leading whitespace at `{#if}` block boundaries during compilation
- Template had `{#if occasion_year} {year}{/if}` — the space was trimmed in the compiled SSR output
- Fix: changed to `{#if occasion_year}{` ${year}`}{/if}` — space inside an expression is untouchable by the compiler
- Fixed on both gift detail and person detail pages

**Bug 3: Disappeared B&H self-package**
- Gift 49 (B&H order, `is_self=1`) was accidentally marked `returned` at 13:19 UTC — likely a fat-finger on the "Mark Returned" button
- `returned` status dropped it from Packages (only shows `ordered/shipped/delivered/wrapped`) and self-people are hidden from Today and People
- No UI path existed to navigate to a returned self-gift
- Fix 1: Added `undoReturnGift()` — restores to the most-advanced timestamp-supported status (`delivered > shipped > ordered > planned`), with "Undo Return" button on the gift detail page
- Fix 2: Packages page now includes a "Returned" section so returned gifts remain reachable

**Bug 4: Skip confirmation (td-4372b9)**
- Too easy to fat-finger the Skip button while scrolling on the Today page
- Both the hero-card "Skip 2026" and inline "Skip" in Coming Up now require a two-tap confirmation (Cancel / Yes, skip)

### Key Technical Insight
Svelte 5's whitespace trimming at block boundaries is a silent gotcha — `{#if cond} text{/if}` compiles to `text` (no leading space). Wrapping the space in a template literal expression (`{` ${text}`}`) is the reliable fix. This differs from Svelte 4 behavior.
