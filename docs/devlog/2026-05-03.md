# 2026-05-03 Devlog

## 23:15 EDT - Person occasion form feedback fix

### Summary
Fixed a "saved but didn't save" UX bug on the admin person edit page where validation errors and success messages were hidden by collapsing `<details>` panels.

### What Was Done
- Diagnosed why a custom occasion ("High School Graduation") appeared not to save: form defaulted to `recurrence="annual"` which the DB CHECK constraint requires `month`/`day` for. Server returned `fail(400, "Month must be 1–12 for annual.")` but the `<details>` wrapper snapped shut on submit, hiding both the error and the user's input.
- Made all three add-occasion `<details>` panels stay open after submit when their scope has form feedback (`open={form?.scope === '...'}`).
- Added inline `Saved.` / `Birthday saved.` / `Occasion assigned.` / `Occasion saved.` confirmation messages, matching the existing Details-form pattern (`role="status"`, AAA contrast).
- Verified `npm run check` clean (0/0 errors/warnings).
- Committed as `7f976b0`, pushed, and deployed to prod via `./scripts/deploy-to-DO.sh`. Health check OK.

### Files Modified
- `src/routes/admin/people/[id]/+page.svelte` — three `<details open=...>` wires + three `ok` message blocks

### Key Technical Insight
Forms inside `<details>` panels need explicit `open=` binding to form state — otherwise the post-submit page reload collapses the panel and hides every error message inside it. This is the second time the project has hit a "form action returned `fail()` but the user couldn't see why" pattern; worth keeping in mind whenever a form is wrapped in collapsible UI.

Separately, confirmed (with the user) that TablePlus over SSH for SQLite is not "live" — every remote SQLite tool must SFTP-copy the `.db` file each refresh because SQLite has no wire protocol. The 1s WAL checkpoint loop ensures the file is fresh when copied, but the refresh itself is still manual. DataGrip/DBeaver/Navicat have the same limitation; "live" SSH was a Postgres/MySQL experience. Documented mentally as a SQLite-vs-Postgres tradeoff, not a tool weakness.
