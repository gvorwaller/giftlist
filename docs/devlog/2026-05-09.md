### Key Technical Insight
**A "missing feature" was a missing entry point.** The Restore action, audit logging, and bidirectional `archiveGift` toggle had been shipping correctly since day one — but no UI surface listed archived rows, so users couldn't reach the action. Lesson: when a user reports "X doesn't work", grep first for the action wiring (it's often there) before assuming a bigger gap. Half the work was confirming what already existed.

The other half — gift history grouped by year — was a UX win the task description hinted at obliquely ("don't duplicate year-over-year"). Worth more than the literal Restore button.
