# 2026-05-08 Devlog

## 18:33 EDT — Self-person dropdown fix on Amazon import review

### Summary
While reviewing an Amazon shipment email, admin couldn't assign it to the "Gaylon — me" self-person — the dropdown silently omitted all self-rows. Patched the loader and added a defense-in-depth privacy guard to the commit path.

### What Was Done
- **Root cause**: `/admin/imports/amazon/review/+page.server.ts:23` called `listPeople({ sort: 'alphabetical' })` with no `includeSelf` flag. The default in `people.ts:58` filters `is_self = 1` rows out — so admin's own self-person was hidden, with no UI signal.
- **Fix**: pass `includeSelf: true, selfOwnerUserId: locals.user.id` so admin sees their own self-row only (manager wouldn't see it even if she could reach /admin).
- **Defense-in-depth**: `commitReviewedRows` in `amazon-import.ts` now revalidates the chosen `personId` via `isPersonVisibleToUser` before binding. Mirrors the td-68804e parity fix on `/app/gifts/new` and `/app/gifts/[id]/edit` — a crafted POST targeting another user's self-row now lands as `failed` with a clear error rather than silently binding.
- **Verification**: `npm run check` 0/0, `npm test` 24/24.
- **Deploy**: commit `b440ee2`, live at gifts.gaylon.photos.

### Files Modified
- `src/routes/admin/imports/amazon/review/+page.server.ts` — `includeSelf: true, selfOwnerUserId: locals.user.id` on the people lookup
- `src/lib/server/jobs/amazon-import.ts` — import `isPersonVisibleToUser`; reject rows that target a non-visible person

### Key Technical Insight
**`listPeople`'s default-deny on self-rows is correct security policy but a UX footgun.** Every admin surface that needs to assign a self-person has to remember to opt in. The Amazon import was the third surface to hit this (after `/app/gifts/new` and `/app/gifts/[id]/edit`, both fixed during td-68804e). Worth grepping for any other `listPeople(` call site that might still be missing the flag — or eventually adding a typed enum (`AudienceScope = 'manager' | 'admin' | 'admin-with-self'`) to make the intent explicit at the call site.
